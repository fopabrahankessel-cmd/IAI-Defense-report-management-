# AICS Report Hub

A Django-based academic report management platform for AICS campuses.

The application supports:

- superadmin and campus admin management
- student and supervisor registration
- bulk assignment of students to supervisors
- one-time upload codes generated by supervisors
- student final report upload
- automatic PDF preview generation
- public report browsing, search, and advanced filtering
- supervisor grading and optional feedback

## Tech Stack

- Python `3.13`
- Django `6.x`
- Pipenv
- SQLite by default
- PostgreSQL optionally through environment variables
- PyMuPDF for automatic PDF preview generation

## Features

- Normal site login for students, supervisors, admins, and superadmin
- Campus-based access control
- Student registration without immediate supervisor assignment
- Separate bulk assignment flow for admins
- Supervisor upload code generation for assigned students
- Student upload form with live AJAX upload-code validation
- PDF viewing inside the platform
- Automatic preview image generation from uploaded PDFs
- Public report repository with search and advanced filters

## Local Setup

### 1. Clone the project

```powershell
git clone <your-repository-url>
cd "Django CA"
```

### 2. Install dependencies

```powershell
pipenv install
```

### 3. Activate the virtual environment

```powershell
pipenv shell
```

If you do not want to open a shell, you can run commands with `pipenv run ...` instead.

### 4. Apply migrations

Run this from the `aics_school` folder where `manage.py` lives:

```powershell
cd aics_school
py manage.py migrate
```

### 5. Create a superuser

```powershell
py manage.py createsuperuser
```

### 6. Start the development server

```powershell
py manage.py runserver
```

Open:

- app: `http://127.0.0.1:8000/`
- Django admin: `http://127.0.0.1:8000/admin/`

## Default Database Behavior

The project uses:

- SQLite automatically if no PostgreSQL environment variables are set
- PostgreSQL if `POSTGRES_DB` is provided

This means a new developer can run the project immediately with SQLite after migrations.

## Environment Variables

You can run the project with defaults, but for a cleaner setup you can configure:

```powershell
$env:DJANGO_SECRET_KEY="change-this"
$env:DJANGO_DEBUG="True"
$env:DJANGO_ALLOWED_HOSTS="localhost,127.0.0.1"
```

To use PostgreSQL:

```powershell
$env:POSTGRES_DB="aics_reports"
$env:POSTGRES_USER="postgres"
$env:POSTGRES_PASSWORD="your-password"
$env:POSTGRES_HOST="localhost"
$env:POSTGRES_PORT="5432"
```

Then run:

```powershell
py manage.py migrate
```

## First-Time App Setup

After creating your superuser:

1. Log in at `/admin/`
2. Create one or more campuses in `Aics centers`
3. Create campus admin users if needed
4. Use the normal site dashboard for most workflows

Important:

- campus creation is currently done in Django admin
- superadmin is the Django `superuser`
- campus admins manage only their own campus

## Main Workflow

### Superadmin

- creates campuses in Django admin
- can manage everything across campuses

### Campus Admin

- logs in through the normal site login page
- registers students
- registers supervisors
- bulk-assigns up to 15 students to a supervisor in one action
- views students, supervisors, and uploaded reports for their campus

### Supervisor

- logs in through the normal site login page
- sees assigned students only
- generates one-time upload codes
- grades reports and adds optional feedback

### Student

- logs in through the normal site login page
- receives an upload code from their assigned supervisor
- uploads one final PDF report
- gets automatic academic metadata based on their profile

## Automatic PDF Preview Generation

Preview images are generated automatically when a report is saved.

This requires:

- `PyMuPDF` installed
- a valid PDF upload

If preview generation fails, the report still saves, but the preview image may be missing.

## Troubleshooting

### `no such table: ...`

You have not applied migrations yet.

```powershell
py manage.py migrate
```

### PDF previews are not showing

Check that:

- the uploaded file is a valid PDF
- `pymupdf` is installed
- the file saved successfully into `media/`

### Reports open but preview is blank

Some PDFs may not render properly depending on file structure. Try another PDF to confirm whether the issue is file-specific.

### Direct PDF download should not be available

Reports are intended to open inside the platform through the controlled PDF route. Direct raw report file access under `/media/reports/...` is blocked in development.

## Notes For Developers

- The custom user model is `core.CustomUser`
- The main domain app is `core`
- Student-to-supervisor assignment is a separate admin action, not part of student registration
- Search and advanced filtering are implemented on the public report repository page
